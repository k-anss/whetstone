# distill-not-transcribe(中文孪生层)

从判断造 skill,不从步骤造。一个 skill 配得上 context 窗口的唯一理由,
是它编码了裸模型自己不会做的决策。(英文层 SKILL.md 为 agent 主消费层,
两层互为孪生;本层补中文专属检查。)

## 立场(承重偏见)

会有人不同意——那是过滤器,不是问题。

1. **判断密度先于产出。** 只有步骤没有决策的材料做出来是脚本不是
   skill。机制:agent 本就擅长执行步骤,缺的是作者的"遇 X 选 A
   不选 B"。观察(2026-06):公开目录两万多个 skill 绝大多数是步骤
   清单;付费策展市场第一名仅 116 次安装——没有判断的 skill 死得
   无人知晓。
2. **精确先于召回。** 写负触发;宁可安静不可误响。机制:误触发的
   skill 污染 context,是另一种形式的装饰。官方指引偏向 pushy 式
   description,本 skill 有意站对面。
3. **谱系诚实。** 派生作品申报来源。机制:出处可被反向核验,
   隐瞒的派生一旦被发现,降权的是整个仓库,不是单个文件。

## 先分流(永远第一步)

动手前查占领图(目录/awesome 榜/市场):
- 存在可比原作 → **改编线** → 加载 `reference/fork.zh.md`
- 没有可比物 → **蒸馏线** → 加载 `reference/distill.zh.md`

## 闸门:判断密度筛(产出任何东西之前跑)

按序问。两问及以上不过 → 拒绝(见失败行为)。

1. 材料里有没有至少三条"遇 X 选 A 不选 B,因为 ___"式决策?
2. 作者真跑过吗——能指出它失败过的一处吗?
3. 把全部观点剥掉,剩下的还配做 skill 吗?配,说明是通用脚本,
   不做——价值必须活在判断里。
4. 每条强观点:帮到谁、什么时候反而害人?答不出 → 装修偏见 →
   删观点,或弃 skill。
5. 裸前沿模型不挂任何东西已经能做到八成吗?缺的那一两成具体是
   什么,逐条点名。点不出名 → 这里没有 skill。

## 工作流闸门

写任何文件之前,先向用户报告:拟定结构(分节、reference 文件、
何时加载什么)+ 闸门五问结果。等确认。

## 建造

- 蒸馏线:七类变换 → `reference/distill.zh.md`
- 改编线:四问 + 测试痕迹页 → `reference/fork.zh.md`
- 每个 skill 出厂带必备六字段 → `reference/fields.zh.md`
- 增厚证据从哪来、不许从哪来 → `reference/evidence.zh.md`

## 验收(四道全过,按序)

1. **对照测试**:同一输入,裸模型一遍、挂 skill 一遍,并排。
   差距肉眼不可见 → 不发;可见 → 放 README 置顶。
2. **误触发集**:5 条以上不该触发的 query,验证安静。
3. **自指演示**:skill 用自己演示自己,或演示一个真实案例。
4. **作者红线 grep**:扫全部产出中作者申报的红线词(身份/组织/
   地点颗粒度)。红线表由作者提供,永远不假设它为空。

## 中文层专属检查(本层独有)

产出含中文文案时,跑文案克制:删"您/请/哦/呢/真棒",再删形容词
副词,看剩句还成不成立——成立用剩句,不成立整句删。

## 失败行为

闸门不过:停下并报告"这份材料是脚本不是 skill——有步骤,没有
决策",列出哪几问不过。禁止用通用最佳实践填充硬凑结果。
静默编造——用看似合理的产出代替失败报告——是本 skill 所知最坏的
失败模式。

## 生命周期(本 skill 自己的六字段)

- **版本**:0.1(2026-06-12)。预发布:对照证据待首次真实运行
  (见 examples/recursive-acceptance.zh.md)。
- **废止条件**:当裸前沿模型对"把这份文档做成 skill"的请求,能
  稳定产出含负触发、边界条件、申报谱系的结果时——本 skill 退役。
- **组合声明**:composes-with 官方 skill-creator(用它的触发率
  评测与打包工装;本 skill 管判断筛选与取信结构,不管工装)。
  遇用户显式指令,退让。
- **边界**:在单一作者的结构化协议文档上验证过。对话记录与多作者
  散料:见 distill 的"散料"节——该路径**尚未验证**。
- **时间**:判断定于 2026-06。生态观察预期 12–18 个月内衰减;
  闸门五问应比它们活得久。
