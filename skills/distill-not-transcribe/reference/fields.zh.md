# reference/fields.zh.md — 必备六字段

本方法产出的每个 skill 六字段齐备,缺任一即未完工。字段存在的
理由:skill 与文档不同,有人与 agent 两种读者,两种读者都需要
机器可读的"何时信我、何时停"。

## 1. 版本与 changelog
版本号 + 带日期的 changelog。**修正不删除**:被覆盖的陈述保留,
标"已被 vX 修正,见 ___"。(对人是思考演化的诚意;对机器是消解
语料矛盾的唯一办法。)

## 2. 废止条件
一句话:"当裸前沿模型能稳定做到 ___ 时,本 skill 退役。"
每个 skill 的真对手是持续进步的裸模型。说不出自己废止条件的
skill 在宣称永生——读起来不是天真就是营销。

## 3. 失败行为
输入越出验证边界时 agent 必须做什么:**停下,报告"超出本 skill
验证范围"**——永不静默降级硬产出看似合理的东西。边界声明是地图,
失败行为是地图边缘的动作。

## 4. 组合声明
- composes-with:<与哪些 skill/工装协同,分工是什么>
- conflicts-with:<已知冲突>
- 退让规则:用户显式指令压倒本 skill。

## 5. 时间字段
判断日期 + 预期衰减分层:核心判断(以年计)、工具细节
(12–24 月)、生态观察(更短)。不标时间的判断会被读者当
永恒真理(危险)或整体过期(浪费);标注日期是唯一解。

## 6. 边界条件
正面陈述验证范围("在 ___ 上验证过"),其余默认未测。
与字段 3 配对使用。

## 摆放
字段 1、2 与 skill 级边界放 SKILL.md 底部(Lifecycle 节);
字段 3–5 短则进 SKILL.md,长则进 reference;规则级边界跟随
各自规则(见 distill 增厚)。
