# examples/recursive-acceptance.zh.md — 递归验收协议与首跑记录

造 skill 的 skill,验收必须递归:用它从源协议重造一个已经存在的
skill,然后 diff。

## 协议

1. 输入:本合集中某已有 skill 的源协议(首跑:填报工具 skill 的
   源协议)。
2. 新会话里端到端跑本 skill:分流 → 闸门 → 结构报告 → 建造 →
   验收。
3. 重造版与已发布版逐节 diff。
4. 读 diff:
   - 重造 ≈ 已发布 → 方法已完整编码;放行。
   - 某处 diff 大 → 那个差值就是还活在作者脑子里、没下到文档的
     判断。每个差值变成对 SKILL.md 或 reference 的一个补丁。
5. 记录于下。本记录同时是本 skill 的测试痕迹页。
