# Self-check Template

Use at least ten substantive lines. A list of headings or repeated “passed” claims is not
enough; name the weakest claim and the disposition it triggered.

```markdown
# chunk_NNN self-check

1. 最可能越过文本的一句话：
2. 最弱因果链及替代解释：
3. 主体、决策者和执行者是否分开：
4. 是否把第三方损失算成主体代价：
5. 是否把未来风险写成已付代价：
6. 是否合并了不同事件：
7. 是否存在A/B重复计数：
8. 引文是否逐字、连续且来自本块：
9. 被拒事件中最可能漏掉的候选：
10. 本次自检触发的 DELETE / DOWNGRADE / SPLIT / REWRITE：
实际读取最后一章：第Y章
```
