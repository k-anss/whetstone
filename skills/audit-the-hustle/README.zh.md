# audit-the-hustle · 审查吆喝

你无法核实一个人是否真在赚钱。你只能读信号——所以输出带把握度的信号,永远不给定论。

## 挂上之后变了什么

**问裸模型**:"这个独立开发者是真在赚钱还是在卖课?"

> "从他持续的发帖和增长的关注来看,他应该是一位真实且成功的独立开发者。"

一个定论,锚在能见度上,底下没有任何可查的东西。

**挂本 skill 问同一个问题**:

> 五类信号逐一扫描标记(有 / 无 / 不可验证)。结论:"70% 像真在经营,30% 像表演"——每条依据标注*清单触发*或*自行推理*。不可验证的收入宣称被明确声明**没有**移动数字。任何外人都核实不了的部分,被点名为"水面之下",交还给你自己的判断,而不是被糊弄过去。

差别不是更多细节,是虚假确定被拿掉了。

## 这是什么

一份纪律文件,用于审查任何公开声称在网上赚钱的人——独立开发者、课程卖家、公开记账的创业者。它强制三件裸模型不会做的事:

1. **信号 + 把握度拆分,永不给定论。** 每次审查以"X% 真经营 / Y% 表演"收尾。永不 100/0。
2. **诚实的够得着边界。** 真实付费、自演、账号背后是谁——水面之下,平实声明"够不着",而不是硬估。
3. **每次结论前的反偏误闸门。** 四个问题,对准模型真实的失效模式:知名名字锚定、"内容即产品"分类错误、叙事美化、未标注的推理冒充证据。

## 这不是什么

- 不是财报核验(用真实财报)
- 不是新闻事实核查
- 不是对私人的背景调查——它只审查*公开*声称在赚钱的人的公开信号

## 安装

把 `audit-the-hustle/` 文件夹复制进你的 agent skills 目录(如 Claude Code 的 `~/.claude/skills/`),或直接把 `SKILL.md` 附进对话。中文孪生文件(`*.zh.md`)内容一致。
