# audit-the-hustle

You can't verify if someone's really making money. You can only read the signals — so report signals with confidence, never a verdict.

## What changes

**Ask a bare model**: "Is this indie hacker really making money or selling a course?"

> "Based on their consistent posting and growing following, they appear to be a legitimate and successful indie hacker."

A verdict, anchored on visibility, built on nothing checkable.

**Ask with this skill loaded**, same question:

> Five signal classes scanned and marked (present / absent / unverifiable). Conclusion: "70% looks like a real operation, 30% looks like performance" — each basis line labeled *checklist-triggered* or *own-reasoning*. The unverifiable revenue claim explicitly did **not** move the number. The parts no outsider can verify are named as below the waterline, handed back to your judgment instead of papered over.

The difference is not more detail. It is the removal of false certainty.

## What it is

A discipline file for auditing anyone publicly claiming to make money online — indie hackers, course sellers, build-in-public founders. It forces three things a bare model won't do:

1. **Signals + confidence split, never a verdict.** Every audit ends in "X% real operation / Y% performance." Never 100/0.
2. **An honest reach limit.** Real payments, self-dealing, who runs the account — below the waterline, stated plainly as unreachable instead of estimated.
3. **An anti-bias gate before every conclusion.** Four questions targeting the model's real failure modes: famous-name anchoring, content-as-product misclassification, narrative beautification, and unlabeled reasoning passed off as evidence.

## What it is not

- Not financial-statement verification (use real filings)
- Not news fact-checking
- Not background checks on private individuals — it audits only the public signals of people *publicly* claiming to make money

## Install

Copy the `audit-the-hustle/` folder into your agent's skills directory (e.g. `~/.claude/skills/` for Claude Code), or attach `SKILL.md` directly to a conversation. Chinese twin files (`*.zh.md`) carry identical content.
