# 示范审查——全虚构

> 以下所有人名、产品、数字、链接模式均为演示虚构。如与真实人物或产品雷同,纯属巧合。

## 请求

"有个独立开发者叫 **'Rilo Vance'**,在公开记账做一个截图转发票的工具 **SnapLedger**,说四个月做到了 $6k MRR。他是真在赚钱还是在卖梦?"

## 第一步——信号扫描(reference/signals.zh.md)

| 类别 | 标记 | 查到的 |
|---|---|---|
| 1. 可验证收入 | 不可验证 | MRR 只出现在帖子和置顶截图里;无支付处理器直连面板,无 API 核验的平台列表 |
| 2. 第三方提及 | 有(弱) | 两条无关联账号发的论坛帖,问 SnapLedger 怎么处理 VAT;一篇博客对比文,作者带分销链接(折价计) |
| 3. 真实差评 | 有 | 一个 6 年账号史的用户抱怨重复扣款和 3 天的客服延迟;Rilo 回帖确认了退款 |
| 4. 运营信号 | 有 | 上月一条 OCR 调优的外包帖;公开 changelog 90 天内更新 11 次 |
| 5. 时间线连续 | 有 | SnapLedger 之前有三年关于自由职业记账自动化的发帖史;纹理一致 |

## 第三步——反偏误闸门(reference/anti-bias.zh.md)

1. 知名名字默认?否——对象无名;证据标准一视同仁。
2. 做产品还是内容即产品?产品。无课程、无付费社群;帖子是 changelog,不是卖方法。(这一问起了作用:一条"我如何做到 $6k"的帖子最初读起来像漏斗;细看链接指向产品,不指向课程。)
3. 在美化吗?$6k 这个数字本身曾把草稿数字推高过一次——已移回;该数字属第 1 类不可验证,不应移动任何东西。
4. 标注见下。

## 输出

**75% 像真在经营,25% 像表演。**

依据:
- 老账号发出的平淡、具体的差评,且公开解决——*清单触发*(第 3 类,最难造假)
- 运营在动:外包招募 + 90 天 11 条 changelog——*清单触发*(第 4 类)
- 三年早于赚钱宣称的连贯历史——*清单触发*(第 5 类)
- $6k MRR 这个数字本身没有移动结论:它不可验证——*清单触发*(第 1 类规则)
- VAT 提问暗示真实司法辖区里的真实用户,摆拍成本高——*自行推理*,按此折算权重

水面之下(未评估):真实流水规模;好评是否有安排。如果你真正想问的是"$6k 是不是真的"——那在水面之下;外部核验能确立的是"一个有真实用户的真实产品存在",不是那个数字的大小。
