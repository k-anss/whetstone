# 信号:五类真实性证据

每类标 **有 / 无 / 不可验证**。永远不要把"不可验证"压成"无"——你本可以查到却没有的证据,是一个信号;任何外人都查不到的证据,不是。

## 1. 可验证收入

算数的:
- 直连支付处理器的公开收入面板(不是自己在页面上敲的数字)
- 有可用购买流程、可见近期活动的在线产品页
- 通过 API 核验、而非截图核验的平台收入列表

不算数的:
- 面板截图(伪造成本极低)
- 帖子或简介里的整数收入宣称
- "收入"实际上是那门教你赚这份收入的课程的售价

## 2. 第三方独立提及

算数的:
- 与对象无关注/分销关系的人做的报道、评测、讨论
- 用户在公开场合主动抱怨或提支持问题
- 在不相关的技术讨论里被当成工具/产品引用

不算数的:
- 对象自己圈子的转发和引用
- 没有可点开原文的"曾被报道"logo 墙
- 分销评测(查链接里的追踪参数)

## 3. 真实差评

算数的:
- 有自身历史的账号发出的具体、平淡的抱怨(扣款故障、客服慢、缺功能)
- 公开讨论的退款纠纷
- 对象认真回应或防御性回应过的批评

不算数的:
- 完美干净的记录(真产品会积累真抱怨;零差评本身就是旗标)
- 小号发的空泛敌意评论(可能是同行,什么也说明不了)

这一类为什么重要:水军可以批量制造好评;批量制造一条跨越数年、无聊而具体的抱怨轨迹,成本高得多。平淡的负面,是最便宜可读、最昂贵可造的信号之一。

## 4. 运营信号在动

算数的:
- 招聘帖、外包需求
- 与在跑的产品一致的域名/基础设施变化(新子域、状态页、changelog)
- 独立工具的流量估计,趋势方向与其宣称一致

不算数的:
- 以上任何一项的单帧快照——信号是*随时间的运动*,不是状态

## 5. 时间线连续

算数的:
- 早于赚钱宣称、纹理一致的发布/出货历史
- 与现在宣称的技能相符的旧内容
- 记录里看得见解释的空档

不算数的:
- 最近醒来、立刻开始叙述成功的账号(诈尸/购入账号模式)
- 被明显修剪过的历史

## 本步骤的输出

一张五行表:类别 | 有/无/不可验证 | 一行写实际查到了什么。这张表喂给 `confidence.zh.md`。不要在这一步下结论。
