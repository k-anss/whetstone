# 把握度:必填的结论格式

## 格式

每次审查的结论必须是这个形状:

> **X% 像真在经营,Y% 像表演/卖课漏斗。**
> 依据:[最影响这个数字的 2–4 条信号,每条标注"清单触发"或"自行推理"]
> 水面之下(未评估):[这次审查看不到的部分]

规则:
- X + Y = 100。永不给 100/0 或 0/100——那是在宣称本框架给不了的核实。
- 强证据可以给高把握度(90/10 没问题)。封顶的是"确定性",不是"倾向性"。
- 百分比陈述的是*证据够到了哪里*,不是预测法庭认定。

## 水面上 vs 水面下

**水面上——框架在这里有用:**
- 五类信号里的任何东西:公开收入核验、第三方提及、差评、运营信号运动、时间线连续
- 用公开记录核对公开宣称

**水面下——框架够不着:**
- 付费墙后面的真实流水
- 销量/好评是否自演
- 账号背后实际是谁在运营
- 改变公开活动含义的私下安排(赞助、未披露的资金)

当用户真正想问的问题在水面之下,用一句平实的话说明,然后停:

> "这在水面之下——真实付费数据和是否自演,外部无法核实。上面的信号已经是一个外人能知道的全部;剩下的靠你自己对这个人和这个圈子的读法。"

不要为水面下的问题生成把握度数字。对够不着的事实编造百分比,是这个 skill 能产出的最坏输出:它读起来像严谨,实际是用空气顶掉了读者自己的判断。

## 校准指引

- 某类信号"有"且强:大幅上调
- "不可验证"的类:不动数字——并明说它没有参与移动数字
- 某类"无"但有平淡合理的解释:小幅移动
- 零差评 + 重度成功叙事:下调(干净记录本身是表演信号)
- 主产品看起来是"关于赚钱的内容"、而不是钱据称来自的那个东西:大幅下调,并把这个模式明说出来
