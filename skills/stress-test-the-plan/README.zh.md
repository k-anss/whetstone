# stress-test-the-plan

讨论自己的计划会让你高估它——而 AI 会附和你。这个 skill 运行
对立角色:找出它死在哪。

## 它做什么

一个面向决策与计划的红队角色。三类拷打:

1. **致命弱点**——能让项目归零的那一个单点,不是次要风险。
2. **隐含假设**——"用户会付费""我能撑六个月":逐条标注为假设,
   各附验证成本。
3. **多轮失效**——即使第一轮成了,1 / 3 / 6 个月后怎么慢慢失效。

不安慰,不盖章。故意悲观。

## 它的差异在哪

不在拷打——裸模型被要求"找漏洞"也能做出大半。增量在边界纪律:

- **你处于情绪低谷时它拒绝运行。** 在糟糕的一天做压力测试是伤害,
  不是帮助。
- **它不重审已上线的项目。** 你已在执行时,它只问"怎么少死",
  绝不问"当初该不该"。
- **它每次都声明自身偏向。** 收尾句固定:这是悲观极端,真实判断
  取它与普通评估之间的中位。
- **叫停即退。** 说"停",角色当轮彻底放下。

## 演示

输入:一个通过电商平台挂单、向本地小微企业卖 AI 辅助文档服务的
计划。

输出(节选——完整运行见 `examples/sample-stress.zh.md`):

```
## 致命弱点
挂单假设平台上存在主动搜索需求;若无人搜索,下游一切都触达不了。

## 隐含假设
1. 买家能分辨你的产出与更便宜的竞品——验证:挂 3 个单、跑 2 周,
   对比询单质量。
...

以上是故意悲观的极端,真实判断请在此和普通评估之间取中位。
```

## 安装

把 `stress-test-the-plan/` 复制进 agent 的 skills 目录。agent 在
触发时加载 `SKILL.md`,按需拉取 `reference/` 文件。
