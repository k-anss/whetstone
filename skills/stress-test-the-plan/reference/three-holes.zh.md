# 三类拷打

三类全跑,按序。每类暴露不同的失败。

## A 类 — 致命弱点

计划最可能死掉的那一个点。不是次要风险:是真正能让项目归零的
那个单点。

- 它通常没被检视过,正因为此前的讨论都在谈"怎么做成",
  没在谈"怎么死"。
- 只点一个。列出五个"致命"弱点等于什么都没排序——挑最可能的
  那个杀招,并说明它为什么排在其余之前。
- 先用一句话说出它,再展开。

## B 类 — 隐含假设

计划当成事实、却从未验证过的断言。常见形态:

- "用户会付费"——这是假设,不是事实。
- "我能撑六个月"——这是假设。
- "这个市场还会在"——这是假设。

每条都要:显式标注"这是假设",并附一个粗略的验证成本(时间、
钱、或一个具体动作)。不带验证成本的假设指控,只是指控。

## C 类 — 多轮失效

即使第一轮成了,它怎么慢慢失效?

- 1 个月后:什么开始滑坡(流失、注意力、精力)?
- 3 个月后:竞争或模仿从哪个方向进来?
- 6 个月后:操作者自己的精力还撑得住吗?这套打法还成立吗?

失效场景必须具体——每个时间点给一个有名有姓的机制,
不许写"事情可能会变难"。

## 输出形态

```
## 致命弱点
[一句话]

## 隐含假设
1. [假设] — 验证成本
2. ...

## 多轮失效
- 1 个月:...
- 3 个月:...
- 6 个月:...

## 结论
[直接说:做 / 不做 / 有条件做——以及为什么]

[中位声明——强制,见 boundaries.zh.md]
```
