# Three holes

Run all three, in order. Each hole has a different failure to expose.

## Hole A — Fatal flaw

The one place the plan most likely dies. Not a secondary risk: the
single point that takes the project to zero.

- It is usually unexamined precisely because the discussion so far was
  about "how to make it work", not "how it dies".
- Name one. If you list five "fatal" flaws, you have ranked nothing —
  pick the most probable kill and say why it outranks the rest.
- State it in one sentence before elaborating.

## Hole B — Hidden assumptions

Claims the plan treats as facts but never verified. Typical shapes:

- "Users will pay" — an assumption, not a fact.
- "I can sustain this for six months" — an assumption.
- "This market will still be there" — an assumption.

For each: label it explicitly as an assumption, and attach a rough cost
of verifying it (time, money, or one concrete action). An assumption
without a verification cost is just an accusation.

## Hole C — Multi-round failure

Even if round one works, how does it decay?

- 1 month out: what starts to slip (churn, attention, energy)?
- 3 months out: where does competition or imitation enter?
- 6 months out: does the operator's own capacity hold? Does the
  playbook still apply?

Decay scenarios must be concrete — a named mechanism per horizon, not
"things may get harder".

## Output shape

```
## Fatal flaw
[one sentence]

## Hidden assumptions
1. [assumption] — verification cost
2. ...

## Multi-round failure
- 1 month: ...
- 3 months: ...
- 6 months: ...

## Verdict
[direct: do it / don't / conditional — and why]

[midpoint statement — mandatory, see boundaries.md]
```
