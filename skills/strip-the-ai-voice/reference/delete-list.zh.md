# 删除清单 — 四类

每类:是什么、词例、动作。示例全部虚构或泛化。

## 1. 敬语与语气助词

词例:您 / 请 / 哦 / 呢 / 呀 / 吧~ / 哟 / 真棒 / 太棒了

动作:删词。如果整句只是为了承载敬语而存在("请您仔细查看哦~"),走删词测试——通常整句删。

注意:祈使指令里的"请"("请勿外传")可能承重。跑一遍测试;删掉"请"改变语义的,保留。

## 2. 形容词副词堆砌

模式:深刻地 / 精妙的 / 全方位、多角度 / 精心创作的 / 强大而简洁的 / 完美地 / 高效地

动作:删修饰,留名词/动词。"精心设计的界面" → "界面"。删完什么都不剩的,说明整句是装饰——删整句(见 `the-test.zh.md`)。

## 3. 解释句

模式:文案解释自己(或工具)在做什么。

- "点击下方按钮以查看详细内容" → 按钮就在那里 → 删句,给按钮写标签。
- "旨在为读者提供一个对照视角" → 直接展示对照,删句。
- "以下是为您整理的要点" → 删句,直接列要点。

动作:删整句。UI 或内容本身就是陈述。

## 4. 替读者下结论句

模式:告诉读者该想什么、该感受什么。

- "这一点至关重要,请重点关注!"
- "助您快速把握全文脉络"
- "相信您一定会感受到它的便捷"

动作:删整句。要点真重要,由结构(标题、位置、强调层级)来承载。

## 操作顺序

1. 第一遍 — 第 1、2 类:删词。
2. 第二遍 — 对动过的句子逐句跑删词测试(见 `the-test.zh.md`)。
3. 第三遍 — 第 3、4 类:删句。
4. 第四遍 — 禁区项(见 `forbidden.zh.md`):见即删。
