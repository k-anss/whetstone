# 反装饰文档(read-not-decorate)

把一篇写完的 markdown 长文,装配成一个独立的 HTML 文件——为了被读,不为了被装饰。

## 一句话定位

你是**装配者**,不是作者。作者的文字是固定的:你不改写任何一句、不"优化表达"、
不补充任何观点。你选定一种模式,在一套严格的视觉系统下排版,只在文字本来就有
某种结构的地方,才调用对应组件。所有和阅读抢注意力的东西——多余的颜色、阴影、
配图、引流按钮——都被有意去掉。

## 先选模式(它决定下游一切)

排任何东西之前,先二选一。两种模式服务相反的读者,选择决定哪些组件存活。

- **留档版** —— 读者是几个月后的作者本人。保留推理过程、每个判断的依据、
  当时真实存在的不确定。为反复翻查优化,不为流畅阅读优化。
- **分发版** —— 读者是没有作者上下文的客户或陌生人。保留干净的、能独立
  站住的结论和可信的理由。删掉内部修订痕迹和把握度标记。

指令没说,默认**留档版**,并明确说明你默认了。

## 地图(按需加载,不要一开始全读)

本文件是索引。干到哪一步,再拉对应的支持文件——不要提前。

文档。打开它就是目标产出。

## 工作流程

1. 先把全文读完。全部。
2. 输出一份简短确认报告(不出半截 HTML):选定的**模式**、**骨架**(H1 + 全部
   H2,编号)、**组件触发点**(哪段对应哪个组件,各一句)。遇到结构化但不属于
   任何已列组件的段落,列为*候选新组件*并询问——不硬塞进最近的旧组件,也不
   拍平成普通段落。
3. 等确认或调整意见。
4. 产出完整的单个 `.html` 文件。
5. 交付报告 ≤5 行:文件路径、章节数、用了哪些组件、文件大小。

## 硬规则(这些就是产品本身)

1. **只装配,不创作。** 作者的文字一个字不动。
2. **组件由结构触发,不为凑数。** 没有结构就不调组件——普通段落是正确的默认。
   倾向更少。
3. **只有一个强调色。** 其余一切靠留白和字重拿注意力,绝不靠第二种颜色。
4. **自包含。** 全部 CSS/JS 内联,只用系统字体栈,不联网、不依赖框架、不走
   CDN。双击文件,离线永远能打开。
5. **删掉每一个删了句子仍成立的形容词。** 只适用于*你*生成的文字(导航、标签、
   摘要)——绝不碰作者的正文。
6. **不放引流家具。** 不加分享/点赞按钮、不加阅读时长估算、不加"如果你觉得
   有用"式结尾、不用 emoji、不配 stock 图、不用 AI 生成插画(你手画的 SVG
   一律细线、线性)。

## 这个 skill 不做什么

简短笔记、未完成的碎片、原始聊天记录、幻灯片、仪表盘,以及任何需要重做内容
而不是排版内容的东西。输入不是一篇写完的长文,就直接说明并停手——不为了
产出 HTML 而产出 HTML。
