# read-not-decorate

**文档是用来读的，不是用来装饰的。**

把一篇写完的 markdown 长文，装配成一个独立的 HTML 文件，供阅读或留档。它在一套严格的视觉系统下排你的文字，只在你的文字本来就有某种结构的地方，才调对应的排版组件。它不改你的句子、不替你补论点、不加任何装饰。

它面向"要被读完"的文档：研究报告、长文、留档的他人文章、技术写作。不做幻灯片、不做仪表盘、不做落地页。

## 适用场景（中文语境）

- 一篇中文长报告，要给半年后的自己留档，连同当时的判断依据和不确定一起留住。
- 一篇要发公众号、又要能独立站住的结论稿，给没有上下文的客户或同行看。
- 留档一篇他人的好文章，保留原文、抹掉装饰，作为离线可读的备份。

两种模式（留档 / 分发）面向相反的读者，开工前先选一种，详见 `reference/modes.md`。

## 演示

`examples/demo.zh.html`(中文)与 `examples/demo.html`(英文)是用这个 skill 做的、讲这个 skill 的同一篇文档。浏览器直接打开（离线、无需构建）即可看到目标产出。

## 安装

三条路径，任选其一：

- **Claude Code 插件**：clone 到你的 skills 目录，命中即加载。
- **手动**：`git clone` 本仓库，把 agent 指向该目录。
- **任意 agent**：把本仓库链接贴进对话，让 agent 直接用。打包是 agent 中立的。

## 结构

`SKILL.md` 是地图，不是全文。它告诉 agent 何时加载哪个支持文件（见英文 README 的表）。先读地图，走到哪一步再拉对应的支持文件。

## 立场（≤5 条）

1. 装配者不重写。文字是作者的，排版是唯一的活。
2. 看到结构才调组件，绝不为凑数而调。
3. 只有一个强调色，其余靠留白拿注意力，不靠颜色。
4. 留档与分发是相反的读者，开工前先选定一个。
5. 删掉每一个删了句子仍然成立的形容词。

不同意第 1 条或第 3 条是预期之内——这是一个立场，不是共识。
