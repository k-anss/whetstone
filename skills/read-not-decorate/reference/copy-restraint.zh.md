# 文案克制

本规范只管**你自己生成的文字**——目录标签、组件 LABEL、折叠面板 summary、
按钮、签名档。**作者的正文你一个字不碰。**

## 三条原则

1. 动词驱动,不用形容词加成。
2. 不解释你做了什么。
3. 不替读者下结论。

## 对照例(LABEL / summary / 按钮)

| 不要这样写 | 这样写 |
|---|---|
| ✨ 太棒了!您已展开本节 | (折叠面板展开不需要提示) |
| 点击下方按钮以查看详细的边界条件分析 | 边界与不适用 |
| 本文核心要点速览,助您快速把握全文脉络 | 目录 |
| 以下是作者精心创作的自案例 | 自案例 |
| 旨在为读者提供一个全方位、多角度的对照视角 | (不写,直接展示对照块) |
| 关键判断:这一点至关重要,请重点关注! | (关键判断本身就够) |

## 对照例(签名档 / meta)

| 不要这样写 | 这样写 |
|---|---|
| 感谢您耐心阅读本文,如有收获,欢迎分享 | (整句删掉) |
| 本文作者某某,资深独立研究者,长期深耕…… | (只留名字) |
| 原创作品,版权所有,转载需注明出处 | (写实际的许可声明那一行) |

## 灰色地带的判断

不确定时,把句子里的形容词、副词全部删掉,看剩下的还成不成立。成立,就用
删后的版本;不成立,说明那句话本身是装饰——整句删。

> "读者将会深刻地感受到机制的精妙之处" → 删形容词副词 → "读者会感受到机制"
> → 残句,不成立 → **整句删。**

中文再多一道:把"您""请""哦""呢""真棒"全删,再删形容词副词,看剩下的成
不成立。

> "您是否确定要清空所有已记录的数据呢?" → 删敬语助词 → "是否确定清空所有
> 已记录的数据" → 再压缩 → "确定清空全部 5 条记录?" ✓

## 全局禁区

不用 emoji。不用动物隐喻(头羊/狼/羊群)。不加分享/点赞/关注家具。不加阅读
时长估算。不加作者长简介。不配 stock 图。不用 AI 生成插画(你手画的 SVG
一律细线、线性)。不写"如果你觉得有用"式结尾。强调色绝不落在装饰元素上——
背景、按钮、边框全部灰阶;只有判断、案例、序号、竖线带强调色。
