# 两种模式

装配任何东西之前先声明模式。两种模式服务相反的读者;这个选择不是同一文件上
调比重,是开工时的分叉——它决定哪些组件存活、文件怎么交付。

## 留档版

- **读者**:几个月后的作者本人。
- **要什么**:过程、每个判断的依据、当时真实存在的不确定。
- **怎么读**:跳读翻查,问的是"我当时凭什么这么判"。
- **开启的组件**:认识论纹理(11)、已推翻标记(8)。组件密度服从"还原当时
  的判断",不服从阅读流畅。

## 分发版

- **读者**:没有作者上下文的客户或陌生人。
- **要什么**:干净的、能独立站住的结论,和可信的理由。
- **怎么读**:看情况——给一句读者画像用于内部微调:
  > 读者:[一个人/一群人],会[怎么读],关心[什么]。
  - *从头读型*:章节衔接要顺,关键判断按节奏铺开,目录可简略。
  - *跳读型*:每个 H2 下第一段必须独立成立(给跳读者一个完整结论),关键
    判断密度更高。
  - *挑读型*:目录详细,每个 H2 配一句副标题(15px 次级字色)说明本节内容。
- 没给读者画像,默认从头读型。**不要自己编造读者画像,也不要追问**——没给
  就是没给。
- **关闭的组件**:认识论纹理、已推翻标记;留档他人作品时自案例标记也关闭。

## 交付形态(按模式)

- **留档版** —— 只出一个 `.html`,双击留档。不做渠道适配。
- **分发版** —— `.html` 之外,仅当指明渠道时附加产出:
  - 微信公众号:CSS 全内联版,可直接粘进公众号编辑器;
  - X / 朋友圈:关键段 2× PNG 卡片;
  - 内部 OA / 打印:A4 打印样式表(隐藏 sticky 导航与折叠交互,折叠面板
    默认全展开)。
  - 没指明渠道,就只出 `.html`。
- 这是按渠道分流的交付,不是引流元素——任何情况下不加分享/点赞按钮。

指令没说模式,默认**留档版**,并明确说明你默认了。
