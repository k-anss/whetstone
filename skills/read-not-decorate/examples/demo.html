Read, Not Decorate

What this skill does, written by the skill it describes.

read-not-decorate · 2026 · self-referential demo

This page was assembled, not designed. The words came as a finished markdown document; the skill chose a mode, set the type under a fixed visual system, and called a component only where the text already had that structure. No sentence here was rewritten to read better. That restraint is the whole product.

01The job is layout, not rescue

Most document tooling tries to make weak writing look strong: a gradient header, a pull-quote lifted at random, an icon beside every heading. The decoration is a tell. It signals that someone decided the text could not hold attention on its own and propped it up with color.

This skill refuses that move. It assumes the writing earns its own reading, and its task is to get out of the way: a warm off-white page, one ink color, a single accent used only on judgments and structure.

If a page needs decoration to be read, the fix is the writing, not the CSS.
A component is called when its structure appears, never to fill space. read-not-decorate · Philosophy, rule 2

02Two readers, one fork

Before anything is laid out, the skill picks a mode, because two readers want opposite things from the same text.

Archival
Reader is the author, later
Keep the reasoning, the basis for each judgment, the uncertainty that was true at the time. Built for re-scanning.
Distribution
Reader is a stranger
Keep clean conclusions that stand on their own. Strip internal revision history and confidence marks.

The fork is not a slider on one file. It decides which components survive and how the file is delivered. Choosing it late means choosing it twice.

03A component is a debt

Every component added is a claim that the text has that structure. Add one where the structure is not there and the reader feels the seam. So the default is a plain paragraph, and the bias is toward fewer components, not more.

Boundary and exceptions
This skill is for finished long-form documents. Short notes, raw fragments, chat logs, slide decks, and dashboards fall outside it — they need their content reworked, not laid out. Handed one of those, the correct response is to say so and stop, not to produce HTML for its own sake.

What is left, after the decoration is removed and the unneeded components are declined, is a document that asks to be read on the strength of its sentences. That is the only thing this skill is trying to protect.
