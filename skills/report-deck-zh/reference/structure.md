# structure.md — Skeleton of a formal Chinese briefing

## Whole-part-whole (总分总)

Every formal briefing closes the loop:

1. **Opening (总)** — one page stating the whole: what period/project this
   covers, the single headline judgment, and the map of sections to come.
2. **Body (分)** — sections, each a parallel-titled unit (see titles.md).
3. **Closing (总)** — restate the whole, now with the delta: what changed,
   what was learned, what comes next. The closing must answer the opening,
   not introduce new material.

A deck that opens with detail or ends on a data page is structurally broken
in this genre, regardless of how good each page is.

## Hierarchical progression

Chinese formal reporting progresses by level, not by free association:

- Level 1: section (章) — one governing judgment per section.
- Level 2: point (条) — 2–4 parallel points supporting the judgment.
- Level 3: evidence — data page or concrete instance under one point.

Rules:
- A lower level never appears without its governing upper level on a
  preceding page.
- Sibling counts stay small and consistent (2–4); seven loose bullets is
  English-deck logic leaking in.
- Progression order inside the body follows the briefing type (below),
  not chronology by default.

## Section patterns by briefing type

| Type | Canonical body order |
|---|---|
| 述职 (duty report) | responsibilities → results by area → problems → plan |
| 总结 (period summary) | overall picture → achievements → shortfalls → next period |
| 立项 (project proposal) | background/need → proposal → resources/risks → ask |
| 复盘 (retrospective) | goal vs outcome → what worked → what failed & why → carry-forward |

If the user's material does not fit one of these, say so and propose a
skeleton explicitly — do not silently force the nearest pattern.

## Page-count discipline

- Opening: 1 page. Closing: 1 page. Each body section: 2–5 pages.
- One judgment per page. A page with two competing messages splits.
