# structure.zh.md — 中文正式汇报的骨架

## 总分总

每份正式汇报都要收口:

1. **开篇(总)**——一页述总:覆盖什么时期/项目、唯一的总判断、
   后续章节地图。
2. **中间(分)**——分章节,每节是一个对仗标题单元(见 titles.zh.md)。
3. **结尾(总)**——回到总,带上增量:变了什么、学到什么、下一步。
   结尾必须回应开篇,不许引入新材料。

开篇直接进细节、或结尾停在数据页的 deck,在这个文体里就是结构坏了,
单页再好也没用。

## 层级递进

中文正式汇报按层级推进,不按自由联想:

- 一级:章——每章一个统领判断。
- 二级:条——2–4 条对仗的支撑点。
- 三级:证据——某条之下的数据页或具体事例。

规则:
- 下级不许在上级判断未出现前先出现。
- 同级数量小而一致(2–4);七条松散 bullet 是英文 deck 逻辑在漏。
- 中间部分的推进顺序按汇报类型走(见下表),默认不按时间流水。

## 各类型的章节模式

| 类型 | 典型中间顺序 |
|---|---|
| 述职 | 职责盘点 → 分领域成果 → 问题 → 打算 |
| 总结 | 总体情况 → 成绩 → 不足 → 下期安排 |
| 立项 | 背景/需求 → 方案 → 资源/风险 → 申请事项 |
| 复盘 | 目标 vs 结果 → 有效之处 → 失效之处及原因 → 带走事项 |

用户材料套不进任何一种时,明说,并显式提出骨架——不许静默硬套
最接近的模式。

## 页数纪律

- 开篇 1 页,结尾 1 页,每个中间章节 2–5 页。
- 一页一个判断。一页里有两个互相竞争的信息,就拆页。
