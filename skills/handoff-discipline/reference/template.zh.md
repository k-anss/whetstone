# template — 交接文档骨架

字段顺序固定。可选字段为空时整节删除,不留占位符。

```markdown
# 交接 · {项目} → {接手上下文}

## 项目一句话
{这是什么,一句}

## 已拍死决定(不许重开)
1. {决定,精确性要紧处原样照抄}
2. ...

## 当前状态
- {事实}
- {事实}

## 待定项(非已拍死——等拍板)
- {条目} — 待定:等 {谁} 拍
- (没有则整节删)

## 战略理由(供理解,不供修改)
{可选;已拍死决定背后的为什么}

## 红线
- {压倒下文一切内容的硬约束}

## 接手方:角色与回执
{从 reference/receiving.zh.md 取用并适配}

## 冲突优先级
用户当场说法 > 本文档 > 任何更早文档。
```

## 字段说明

- **项目一句话** — 接手方零上下文;在它读到后文之前,这一行就是它
  的全部世界模型。让它承载名词("一个 skills monorepo"、"一条数据
  管线"),不承载情绪。
- **已拍死决定** — 承重段。编号,一行一条。凡接手方必须原样复现的
  精确字符串(账号名、命名、版本号)都写进来。回执里接手方回显的
  就是这份清单。
- **当前状态** — 只写可验证的事实。见 layers.zh.md。
- **待定项** — 每条都写明由谁拍板。匿名的待定项在下游无法收口。
- **战略理由** — 可选。膨胀之前先砍;被理由淹没的接手方,会把理由
  当成邀约。
- **红线** — 即使接手方收到用户指令也需显式覆盖才能动的约束(隐私、
  范围、发布规则)。少而绝对。
- **冲突优先级** — 永远是最后一行,永远这个顺序。文档必须把自己
  置于当场用户之下,否则它就变成了它本要防止的那种被冻结的判断。
