# Skill 实测记录采集

> 用途:在真实项目里实测第三方 skill 时,当场记录测试事实,导出结构化
> 文本供后续测评写作与归档解析。
> 本表是 `missing-not-zero` 的演示输入;`demo.zh.html` 是据它装配的成品。

## 公共头

| 字段 | 类型 | 规则 |
|---|---|---|
| 日期 | date | 必填 |
| 项目 | text | 必填 |
| 记录人 | text | 必填 |

## 模块一 · 实测记录

### 采集项-01 · 单次实测

> 同一 skill 一天内可多次实测,每次一条。该触发而未触发的情况也记在
> 本模块(算一次失败的实测)。

| 字段 | 类型 | 规则 |
|---|---|---|
| skill 名称 | text | 必填 |
| 来源(仓库/链接) | text | |
| 测试输入概述 | textarea | 必填 |
| 未挂 skill 表现 | textarea | |
| 挂 skill 表现 | textarea | |
| 判定 | select [留/改/扔/待定] | 必填 |
| 改动点 | textarea | 条件必填:判定 = 改 |

## 模块二 · 误触发检查

### 采集项-02 · 反向用例

> 本模块只记「不该触发的指令」。该触发而未触发的,归模块一。
> (归属规则写明,防跨模块歧义。)

| 字段 | 类型 | 规则 |
|---|---|---|
| 不该触发的指令 | textarea | 必填 |
| 实际是否触发 | select [未触发/触发了/不确定] | 必填 |
| 备注 | textarea | |

## 格式化输出统一规则

按 skill 默认格式(见 `reference/output-format.zh.md`):`===` 分隔、
末条后也加 `===`、空记录两种(本项无发生 / 本项未采集)。
