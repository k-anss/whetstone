# 输出格式:导出即产品

写记录组装与导出区时读本文件。

工具的终点不是「数据存下来了」,是「数据干净地交到下一道工序」:贴给
agent、被脚本解析、被人读。导出通道才是全部目的;其余一切是围着它搭的
脚手架。

## 一条真实记录

```
===
[采集项-NN] {采集项中文名}
日期:YYYY-MM-DD
项目:{项目}
记录人:{记录人}
---
{字段名 1}:{值}
{字段名 2}:{值}
---
可补充:{若空则省略此行}
照片:[{文件名,空则省略此行}]
===
```

(公共头几行——上例的日期/项目/记录人——以字段表定义的公共头为准。字段表
可整体覆盖此布局;字段表赢。)

## 两种空记录(绝不是一种)

按了 **[此项无发生]**:

```
===
[采集项-NN] {采集项中文名}
日期:YYYY-MM-DD
项目:{项目}
记录人:{记录人}
---
本项无发生
===
```

按了 **[未采集/不确定]**:

```
===
[采集项-NN] {采集项中文名}
日期:YYYY-MM-DD
项目:{项目}
记录人:{记录人}
---
本项未采集(不确定是否发生)
===
```

*为什么是两种格式:*下游解析者必须不靠人脑补,就能把「确实没有」与
「没人查过」分开。合并成一行「无数据」,逼分析者去猜,而两种猜法各错
一处:缺失当零会低估,零当缺失会派人去复查一件从没发生的事。

## 分隔符与可解析性

- 记录之间用 `===` 分隔——**最后一条记录之后也加 `===`**,下游
  `split("===")` 不需要特判。
- `<pre>` 预览用等宽字体栈——视觉提示:这段文本下一站是机器,不是散文。
- **导出文本里零评论。**不加「以上数据已采集」,不加合计,不加签名。
  导出只含记录,别无其他。

## 导出控件

- **复制全部** —— 全文写入剪贴板;按钮临时变强调色、文字变「已复制」,
  2 秒恢复。
- **下载为 .txt** —— 文件名 `采集_{日期}_{项目slug}.txt`。
- **清空** —— 全工具唯一的二次确认:「确定清空全部 N 条记录?」。

## 存储

- 记录持久化在 `localStorage` 的 `K_form_records_{字段表slug}`;刷新不丢;
  只有显式 [清空] 才删。
- 公共头单独持久化(`K_form_header_{字段表slug}`),明天打开自动填回。
