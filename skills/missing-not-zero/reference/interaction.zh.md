# 交互:骨架、三按钮、校验

搭页面结构与行为时读本文件。

## 固定骨架(所有工具都长这样)

```
[顶部固定区]
  标题 + 一句副标题(取字段表;没有就不写)
  [公共头]:本轮共用字段(如 日期/项目/记录人)
    —— 折叠面板,首次打开展开,必填项填完自动收起;收起态显示一行摘要
      (「2026-06-12 · 项目A · 记录员1」);点摘要可重新展开修改。
    —— localStorage 持久化,下次打开自动填回。

[模块切换标签栏](sticky 于公共头之下)
  模块一 | 模块二 | …
  active 标签:强调色下划线。切换过渡 ≤150ms,不许更花。

[当前模块的采集项卡片]
  ┌─ 采集项 NN · 标题 ───────────────────────────────┐
  │ 字段 1: [输入框]                                   │
  │ 字段 2: [选择器]                                   │
  │ …                                                 │
  │ [此项无发生] [未采集/不确定] [追加此条到输出]        │
  └───────────────────────────────────────────────────┘

[输出区](sticky 底部,默认折叠)
  折叠:「▾ 已记录 N 条 · 展开预览」
  展开:<pre> 结构化文本 </pre>
        [复制全部] [下载为 .txt] [清空]
```

## 三按钮(心脏——语义是承重的)

每张采集项卡片底部恰好三个按钮。前两个记录的是*数据的缺席*,且**不可
互换**:

**[此项无发生]** —— 这件事本轮确实没有发生(如「本工序今日无拆装」)。
- 向输出区追加一条「本项无发生」记录(格式见 `output-format.zh.md`)。
- 卡片显示状态「已记录:无发生」,字段全部 disabled。
- 给 5 秒**撤销**;撤销后字段恢复可编辑。

**[未采集/不确定]** —— 可能发生了,但本人没查到/不掌握。
- 追加一条「本项未采集(不确定是否发生)」记录。
- 同样的 disable + 5 秒撤销。
- 两个无数据按钮视觉同级、次级字色,不许抢 [追加] 的主操作位。

**[追加此条到输出]** —— 一条真实记录。
- 先校验必填 + 条件必填。不通过:缺的字段上警告色边框,顶部一条通知
  「还有 N 项必填未填」(3 秒自动消失)。不用 alert(),不用弹窗。
- 通过:按字段表的输出规则组装记录,追加到输出区,卡片显示
  「已记录 · {第一个文本字段的值}」,然后**清空字段**——同一采集项一天内
  可能记录多次。

不加任何其他按钮。没有「保存草稿」,没有「分享」,没有「上传」。

## 为什么两个无数据态不能合并

合并成一个按钮,意味着下游解析对两种不同的事实只收到一个符号。缺失当零
会低估;零当缺失会触发不必要的复查。这道拆分必须发生在**填报当场**——
事后不可逆。这就是本 skill 存在的理由;若字段表要求合并两态,停下报告
(失败行为,见 SKILL.zh.md)。

## 报错

- 顶部一条横向通知,文字如「还有 3 项必填未填」,数字用强调色,3 秒
  自动消失。
- 不用弹窗、不用 alert/confirm——唯一例外:[清空] 二次确认一次,
  「确定清空全部 N 条记录?」,按钮排版避免误点。
- 未填的必填字段:警告色边框,聚焦后恢复。

## 持久化

- 只用 `localStorage`。键:
  - 公共头:`K_form_header_{字段表slug}`
  - 记录:`K_form_records_{字段表slug}`
- 刷新不丢;只有显式 [清空] 才删记录。
- 无服务器、无同步,导出只有复制/下载两条路。
