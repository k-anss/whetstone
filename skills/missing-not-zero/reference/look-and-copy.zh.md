(字段表里写明的字段名、采集项名、模块名:你一个字不动。)

## 文案克制(三原则)

1. **动词驱动,不用形容词加成。**
2. **不解释工具在做什么。**
3. **状态变化用活得下来的最短的词。**

| ✗ 不要 | ✓ 要 |
|---|---|
| 「✨ 太棒了!您已成功记录本条数据」 | 「已记录」 |
| 「请您仔细检查所有必填项后再次尝试提交」 | 「还有 3 项必填未填」 |
| 「数据已成功复制到您的剪贴板,请前往目标位置粘贴」 | 「已复制」 |
| 「您是否确定要清空所有已记录的数据呢?此操作不可撤销哦~」 | 「确定清空全部 5 条记录?」 |
| 「当前没有任何数据,请先开始填写」 | (不写;空状态自明) |
| 「本工具旨在帮助您高效完成 XX 数据采集」 | (副标题写「用于 XX 采集」,或整行删) |
| 「请输入您本次工作的项目名称(必填)」 | 「项目 *」(标签+星号,不写「请输入」) |

## 中文专属删词算法(英文层没有的一道)

不确定一句文案要不要,按序执行:

1. 删光「您 / 请 / 哦 / 呢 / 啦 / 哟」与一切敬语助词;
2. 再删光形容词、副词(「成功地」「完美」「仔细」「真棒」);
3. 看残句:
   - 还成立 → 用残句(可再压缩:「是否确定清空所有已记录的数据」→
     「确定清空全部 5 条记录?」);
   - 不成立 → 这句话本身是装饰,问一遍「这条提示需要存在吗」——
     多数不需要,**整句删**。

例:「当前没有任何数据,请先开始填写」→ 删后剩「没有数据,先填写」→
UI 上方就是输入框,这是废话 → 整句删,空状态留白。
