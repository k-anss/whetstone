# 字段映射:字段表标记 → HTML,以及归属歧义闸

把字段表翻成输入控件时读本文件;且**每次**生成前——先跑下面的歧义闸。

## 跨模块归属歧义闸(先于任何 HTML)

动手前对字段表问一句:**现实中会不会有同一件事,可能落进多个模块?**
逐对走一遍模块,设想填报人实际会遇到的事。

会——停下,报告。给用户两条修法:
1. 合并重叠的模块;或
2. 一条显式归属规则,写进工具副标题或采集项标题
  (如「按*主要动作*所属模块填」)。

**禁止**自己挑一个归属硬产出。

*field 观察:*见过同一类事同时沾几个模块:不同的填报人选了不同模块,下游
按模块统计直接算错——而表面上每格都「填了」,错误无声。「无声」是关键词:
这种失败永远不报错。唯一能拦住它的位置就是这里,在工具存在之前。

*边界:*在按功能/工序划分模块的表内验证过。按时间线或按对象划分的表,
歧义形态可能不同——未测;遇到先声明。

## 标记 → 实现对照表

| 字段表标记 | HTML 实现 |
|---|---|
| `text` | `<input type="text">` |
| `textarea` | `<textarea>`,初始 2 行,自动撑高,不限最大 |
| `number` | `<input type="number" min="0">`(整数;min 防负值) |
| `time` | `<input type="time">` |
| `date` | `<input type="date">` |
| `datetime` | `<input type="datetime-local">` |
| `select [A/B/C]` | 原生 `<select>`——绝不自造下拉 |
| `select [A/B/C/其他] + text` | select;选「其他」时旁出 text 输入 |
| `multi-select [A/B/C]` | 每个选项一个 checkbox |
| `必填` | 标签后强调色 `*` + required 属性 |
| `条件必填` | JS 监听触发字段;被依赖字段触发才出现,出现即必填 |
| 标注「隐藏,不在 HTML 出现」的采集项 | 完全不渲染 |

## 条件必填,说精确

「中断时长 > 0 时必填中断原因」意味着:
- 条件不成立时,被依赖字段**不存在**(不是 disabled,是不渲染);
- 它一出现,即必填;
- 条件失效,它消失,其值从待提交记录中丢弃。

每一对条件依赖都要列进结构确认报告(SKILL.zh.md 闸门 3)——逻辑跑起来
之前,用户必须先看到它。

## 这里你永远不做的事

- 加字段表没定义的字段——哪怕「显然该有」。
- 扩 select 选项——哪怕只加一个。
- 因为「另一种更合适」就换字段类型。
- 静默二选一来消解字段表里的矛盾。矛盾 → 停下报告(失败行为)。

移动端注:输入框 16px,手机浏览器不自动放大;time/date 调系统选择器;
短字段(time/date/select)可两列并排,其余占满整行。
