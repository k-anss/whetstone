# missing-not-zero

**缺失不是零。**

把一份字段标准化表,装配成单文件 HTML 填报工具——浏览器打开、现场反复
填报、导出结构化文本。它存在的理由是一条多数表单生成器跳过的纪律:每个
采集项必须落在**三种**状态之一——*已记录*、*此项无发生*、*未采集/不确定*
——因为「确实没这事」和「没人查过」一旦共用一个按钮,下游分析就再也分不开
它们了。

## 十秒清单(本 skill 强制、默认生成不会有的东西)

1. **每个采集项两个无数据按钮**,绝不是一个「无」——导出格式也是两种。
2. **遇跨模块归属歧义就停。**同一件事可能落进两个模块时,agent 在动手前
   报告,而不是交付一个会无声算错的 schema。
3. **拒绝连贯分析进表单。**论证链塞进格子,导出来是一堆互不相干的碎片;
   agent 会把它转给阅读页类 skill。
4. **导出即可解析**:`===` 分隔、*末条后也有一个*、数据里零评论。
5. **字段表是法律**:不加字段、不扩选项,矛盾报告而不是静默二选一。

以上每条在 skill 正文里都配机制与 field 观察作底——不是口味。

## 演示

`examples/field-spec.zh.md` 是真实输入;`examples/demo.zh.html` 是本 skill
据它装配的成品。刻意自指:这是一个记录 skill 实测的工具——由本 skill 造,
可用来测本 skill 这类东西。任意浏览器双击打开,离线,零构建。

## 安装

- **Claude Code 插件** —— clone 进 skills 目录,agent 命中即加载。
- **手动** —— `git clone` 本仓库,把 agent 指向本目录。
- **任意 agent** —— 把本目录链接贴进对话,让它直接用。结构是 agent 中立的。

## 结构

`SKILL.zh.md` 是地图,reference 按需加载:

| 正在做 | 读 |
|---|---|
| 骨架、三按钮、校验、撤销 | `reference/interaction.zh.md` |
| 字段标记 → 控件;条件字段;归属歧义闸 | `reference/field-mapping.zh.md` |
| 导出记录、两种空格式、存储 | `reference/output-format.zh.md` |
| 配色、字号,及 agent 自己写的任何文案 | `reference/look-and-copy.zh.md` |

全部文件有英文孪生(`*.md`、`demo.html`)。中文层带英文层不重复的中文
删词算法(删「您/请/哦/呢」那道)。

## 自己跑一遍 before/after

开一个干净会话,把 `examples/field-spec.zh.md` 交给不挂 skill 的 agent,
要一个单文件采集工具;再挂上本 skill 做同一件事。拿两个产出对照上面的
十秒清单。要是看不出差别——告诉我,那正是本 skill 废止条件被触发的样子。

**不该**触发本 skill 的指令(误触发反向用例):

1. 「把这份 markdown 报告排成阅读页 HTML」→ `read-not-decorate`
2. 「帮我写季度分析报告」
3. 「做一个带用户登录的报名系统」
4. 「给客户发一次性的满意度问卷」
5. 「用 React 做个数据 dashboard」
6. 「把这篇长文论证拆成要点」
7. 「做个 Excel 模板来统计」
8. 「解析这份导出的 txt 数据」
