# backtest-your-framework

别问 AI 你的框架好不好——它会同意。
冻结它的上下文,让历史来打分。

一个把"框架验证"做成盲测实验的 skill。你带来一个判断框架(投资
筛选、招聘评估、风险打分——任何按规则下结论的决策系统)和若干你
知道结局的历史案例。本 skill 强制执行一套协议:AI 在上下文冻结到
时间 T 的条件下判断每个案例,提交判断,然后才看到真相。

## 为什么存在

几乎所有"用 AI 验证框架"的尝试,都毁在两个失败模式上:

1. **乐观附和。** 被征求意见时,模型站在你的乐观里作答。那些赞许
   流畅而无价值。
2. **训练数据作弊。** 知名案例的结局就在模型权重里。一个可识别的
   名字,会让判断从向前推理翻转成向后合理化——不可见地。

红队、评测类 prompt 市面上不少。几乎没有一个把这两个自欺源做成
强制的实验协议。这一个做了。

## 前后对照

**不挂 skill** ——"我这套机会筛选器好不好?"

> 模型点评设计、建议补两条标准、说整体结构"看起来扎实"。
> 没有任何案例被测试。没有任何东西被学到。

**挂 skill** ——同一个问题:

> 模型拒绝用意见打分。它带你走案例设计(冻结时间、改名、中性
> 描述),回显脱敏清单等你确认,跑一次带把握度数字和失败模式的、
> 已提交的盲判,再与揭示的真相对比,把结果记入偏差表。
> 如果它发现自己知道结局,它会中止并告诉你。

## 内容

```
SKILL.md            地图:三条承重立场、工作流闸门、失败行为
                    (中文孪生:SKILL.zh.md)
reference/
  three-laws.md     时间冻结 / 信息隔离 / 先判后揭
  protocol.md       五阶段流程及各阶段产出物
  anti-cheating.md  两个自欺源,外加用户侧泄露
  bias-table.md     跨案例汇总的偏差表模板
examples/
  sample-case.md    一次完全虚构的端到端示范
```

所有文件均有 `.zh.md` 孪生版。

## 安装

```
git clone https://github.com/k-anss/whetstone
cp -r whetstone/skills/backtest-your-framework ~/.claude/skills/
```

或把文件夹复制到你的 agent 加载 skill 的任何位置。

## 适用范围

仅适用于判断型框架。不适用于价格数据上的量化策略回测,也不适用于
缺乏可靠历史案例的领域。
